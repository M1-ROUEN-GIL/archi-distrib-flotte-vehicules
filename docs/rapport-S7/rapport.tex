\section{Observabilit\'{e} Compl\`{e}te}
% ============================================================

L'architecture distribu\'{e}e de la plateforme a \'{e}t\'{e} enrichie d'une stack d'observabilit\'{e} compl\`{e}te permettant de surveiller la sant\'{e} du syst\`{e}me et de diagnostiquer les problèmes en temps r\'{e}el.

\subsection{Tra\c{c}age Distribu\'{e} avec Jaeger}

Chaque microservice (Java et Node.js) intègre l'\textbf{agent OpenTelemetry} pour collecter et exporter des traces distribu\'{e}es. Cela permet de visualiser le cheminement complet d'une requ\^{e}te, de la Gateway GraphQL jusqu'aux services backend et aux bases de donn\'{e}es.

\begin{itemize}[leftmargin=*]
  \item \textbf{Instrumentation automatique} : Utilisation du \texttt{opentelemetry-javaagent.jar} pour les services Java et du SDK OpenTelemetry pour Node.js.
  \item \textbf{Propagation de contexte} : Les en-t\^{e}tes de trace sont transmis lors des appels gRPC et des messages Kafka.
\end{itemize}

\subsection{Collecte de M\'{e}triques avec Prometheus}

Un serveur \textbf{Prometheus} centralise les m\'{e}triques techniques et m\'{e}tiers du cluster.
\begin{itemize}[leftmargin=*]
  \item \textbf{Spring Boot Actuator} : Expose les m\'{e}triques JVM, HTTP et Kafka via le endpoint \texttt{/actuator/prometheus}.
  \item \textbf{NestJS Metrics} : Int\'{e}gration de m\'{e}triques Prometheus personnalis\'{e}es pour le service de localisation.
  \item \textbf{Kafka Exporter} : Surveillance du lag des consommateurs et du d\'{e}bit des topics.
\end{itemize}

\subsection{Dashboards Grafana Personnalis\'{e}s}

Nous avons conçu des tableaux de bord \textbf{Grafana} pour offrir une visibilit\'{e} multi-niveaux :
\begin{enumerate}[leftmargin=*]
  \item \textbf{System Health} : CPU, M\'{e}moire, Taux d'erreurs par service.
  \item \textbf{Business Metrics} : Nombre de v\'{e}hicules en mouvement, alertes actives par s\'{e}v\'{e}rit\'{e}, taux de disponibilité de la flotte.
  \item \textbf{Kafka Traffic} : Visualisation des flux de messages entre les services.
\end{enumerate}

\section{Interface de pilotage des Alertes}

Une nouvelle page \texttt{/alerts} a \'{e}t\'{e} int\'{e}gr\'{e}e au frontend pour permettre aux op\'{e}rateurs de visualiser et d'acquitter les alertes en temps r\'{e}el.

\begin{infobox}
\textbf{Choix d'architecture :} Contrairement aux autres modules m\'{e}tiers, la gestion des alertes a \'{e}t\'{e} impl\'{e}ment\'{e}e directement au sein de l'application \textbf{Shell} et non comme un micro-frontend (MFE) s\'{e}par\'{e}. Ce choix permet d'\'{e}viter la complexit\'{e} disproportionn\'{e}e qu'induirait la cr\'{e}ation d'un d\'{e}p\^{o}t et d'un pipeline de d\'{e}ploiement d\'{e}di\'{e} pour un composant aussi transverse et l\'{e}ger.
\end{infobox}

Cette interface consomme le flux de messages du \texttt{events-service} via GraphQL et permet un pilotage r\'{e}actif de la flotte directement depuis le cockpit central.

\section{Tests de Charge et Performance (K6)}
% ============================================================

Pour valider la robustesse de l'architecture face \`{a} une montée en charge, nous avons utilis\'{e} \textbf{K6}.

\subsection{Sc\'{e}narios de Tests}

Plusieurs sc\'{e}narios ont \'{e}t\'{e} mod\'{e}lis\'{e}s dans le dossier \texttt{load-tests/} :
\begin{itemize}[leftmargin=*]
  \item \textbf{Flux GPS Massif} : Simulation de 500 v\'{e}hicules envoyant simultan\'{e}ment leurs positions via gRPC.
  \item \textbf{Consultation Dashboard} : Requ\^{e}tes GraphQL complexes simulant l'acc\`{e}s de 50 managers simultan\'{e}s.
  \item \textbf{Stress Test Kafka} : Injection rapide d'alertes pour v\'{e}rifier la capacit\'{e} de traitement du \texttt{events-service}.
\end{itemize}

\subsection{R\'{e}sultats et Optimisations}

Les tests ont mis en \'{e}vidence la n\'{e}cessit\'{e} de configurer des \textit{HPA (Horizontal Pod Autoscalers)} sur la Gateway et le service de localisation. Le passage \`{a} une communication asynchrone via Kafka pour les alertes a permis de maintenir une latence stable sous charge.

% ============================================================
\section{D\'{e}ploiement Final avec Helm}
% ============================================================

Le d\'{e}ploiement a \'{e}t\'{e} uniformis\'{e} via \textbf{Helm} pour \^{e}tre \og prod-ready \fg{}.

\subsection{Organisation des Charts}

\begin{itemize}[leftmargin=*]
  \item \textbf{\texttt{fleet-infra}} : Gère les dépendances (PostgreSQL, Kafka, Keycloak, Prometheus, Jaeger).
  \item \textbf{\texttt{fleet-app}} : Déploie l'ensemble des microservices et le frontend.
  \item \textbf{\texttt{fleet-observability}} : Configure Grafana et les dashboards.
\end{itemize}

\subsection{Configuration Ingress et S\'{e}curit\'{e}}

L'acc\`{e}s externe est g\'{e}r\'{e} par un Ingress Controller (Traefik ou Nginx) avec :
\begin{itemize}[leftmargin=*]
  \item Isolation des réseaux via des \textbf{Network Policies} Kubernetes.
  \item Gestion des secrets via des \texttt{Secret} Kubernetes (inject\'{e}s au runtime).
  \item \textit{Readiness} et \textit{Liveness probes} configur\'{e}es pour chaque service.
\end{itemize}

\section{Documentation et Finalisation}
% ============================================================

\subsection{Documentation Technique}

La documentation a \'{e}t\'{e} centralis\'{e}e et mise \`{a} jour :
\begin{itemize}[leftmargin=*]
  \item \textbf{README.md} : Guide d'installation rapide (Docker \& Kubernetes).
  \item \textbf{API Docs} : Documentation OpenAPI pour les services REST et sch\'{e}ma GraphQL auto-g\'{e}n\'{e}r\'{e}.
  \item \textbf{ADR (Architecture Decision Records)} : Justification des choix technologiques (Kafka, Micro-frontend, gRPC).
\end{itemize}

\subsection{Pr\'{e}paration de la D\'{e}monstration}

Pour le rendu final, un sc\'{e}nario complet a \'{e}t\'{e} pr\'{e}par\'{e} :
\begin{enumerate}
  \item Initialisation des donn\'{e}es (Conducteurs, V\'{e}hicules).
  \item Lancement d'une simulation GPS avec d\'{e}passement de vitesse.
  \item Visualisation de l'alerte sur le frontend.
  \item Consultation des traces dans Jaeger pour analyser le flux de l'alerte.
\end{enumerate}
